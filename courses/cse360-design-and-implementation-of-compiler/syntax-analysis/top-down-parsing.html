# Top-Down Parsing

The section begins with a general form of top-down parsing, called recursive-descent parsing, which may require backtracking to find the correct AAA-production to be applied. Later we'll introduce predictive parsing, a special case of recursive-descent parsing, where no backtracking is required. Predictive parsing chooses the correct AAA-production by looking ahead at the input a fixed number of symbols, typically we may look only at one (that is, the next input symbol).

# Recursive-Descent Parsing

Consider the grammar

S→cAdA→ab∣a\begin{matrix} S & \rightarrow & cAd \\ A & \rightarrow & ab \; \vert \; a \\ \end{matrix}SA​→→​cAdab∣a​

To construct a parse tree top-down for the input string w=cadw = cadw=cad, begin with the starting node SSS, and the input pointer pointing to the first character from left to right of the input, i.e., ccc.

  1. Since SSS has only one production, we use it to expand SSS and obtain the tree of the lhs of the figure below.
  2. The leftmost leaf, labeled ccc, matches the first symbol of input www, so we advance the input pointer to aaa. Now, use the first alternative A→abA \rightarrow abA→ab to obtain the tree of the middle of the figure below.
  3. Since bbb does not match ddd, we report failure and go back to AAA and retract the input pointer to seek for another alternative producing a match. The second alternative A−>aA -> aA−>a produces the tree of rhs of the figure below. The leaf aaa and ddd match to the second and the third symbols of www respectively, we halt and annouce successful completion of parsing.

Noted that a left-recursive grammar can cause a recursive-descent parser, even one with backtracking, to go into an infinite loop. That's why we've talked about the elimination of left recursion. Also, the backtracking is not an efficient way to construct a parse tree, thus we shall see how to predict the right alternative below.

# FIRST Set

# Why FIRST

Consider the production rule

S→cAdA→bc∣a\begin{matrix} S & \rightarrow & cAd \\ A & \rightarrow & bc \; \vert \; a \\ \end{matrix}SA​→→​cAdbc∣a​

Assume the input string is cadcadcad. Scanning from left to write, upon reading ccc, we know that S→cAdS \rightarrow cAdS→cAd may be applicable but only if AAA produces a string led by aaa. Therefore, we need to know what comes first in AAA, i.e. FIRST(A)FIRST \lparen A \rparenFIRST(A).

In this case, FIRST(A)={b,a}FIRST \lparen A \rparen = \{b, a\}FIRST(A)={b,a}, so we can tell that AAA is applicable and use the production rule A→aA \rightarrow aA→a to build the following parse tree.

# Rules for FIRST set

To compute FIRST(X)FIRST(X)FIRST(X) for all grammar symbol XXX, apply the following rules until no more terminals or ϵ\epsilonϵ can be added to any FIRST set.

  1. If XXX is a terminal, then FIRST(X)={X}FIRST(X) = \{X\}FIRST(X)={X}.
  2. If XXX is a nonterminal and X→Y1Y2⋯YkX \rightarrow Y_1Y_2 \cdots Y_kX→Y1​Y2​⋯Yk​ is a production for some k≥1k\geq 1k≥1
    • Place aaa in FIRST(X)FIRST(X)FIRST(X) if for some iii, aaa is in FIRST(Yi)FIRST(Y_i)FIRST(Yi​), and ϵ\epsilonϵ is in all of FIRST(Y1),⋯,FIRST(Yi−1)FIRST(Y_1), \cdots , FIRST(Y_{i-1})FIRST(Y1​),⋯,FIRST(Yi−1​), i.e., Y1⋯Yi−1⇒∗ϵY_1 \cdots Y_{i-1} \xRightarrow{*} \epsilonY1​⋯Yi−1​∗​ϵ.
    • Add ϵ\epsilonϵ to FIRST(X)FIRST(X)FIRST(X) if ϵ\epsilonϵ is in FIRST(Yj)FIRST(Y_j)FIRST(Yj​) for all j=1,2,⋯,kj = 1, 2, \cdots ,kj=1,2,⋯,k.
  3. If X→ϵX \rightarrow \epsilonX→ϵ is a production, then add ϵ\epsilonϵ to FIRST(X)FIRST(X)FIRST(X).

Elaboration on the Rules

  1. For example, suppose A→a∣(B)A \rightarrow a \; \vert \; \lparen B \rparenA→a∣(B), then FIRST(A)={a,(}FIRST(A) = \{ a , \lparen \}FIRST(A)={a,(}, where aaa and (\lparen( are terminals
  2. For example, everything in FIRST(Y1)FIRST(Y_1)FIRST(Y1​) is surely in FIRST(X)FIRST(X)FIRST(X). If Y1Y_1Y1​ does not derive ϵ\epsilonϵ, then we add nothing more to FIRST(X)FIRST(X)FIRST(X), but if Y1⇒∗ϵY_1 \xRightarrow{*} \epsilonY1​∗​ϵ, then we add FIRST(Y2)FIRST(Y_2)FIRST(Y2​), and so on.
  3. The rule should be self-explanatory.

# LL(1) Grammars

Predictive parsers, that is, recursive-descent parsers needing no backtracking, can be constructed for a class of grammars called LL(1).

  1. The first "L" in LL(1) stands for scanning the input from left to right,
  2. The second "L" for producing a leftmost derivation
  3. The "1" for using one input symbol of lookahead at each step to make parsing action decisions.

# Definition of LL(1) grammars

A grammar GGG is LL(1) if and only if whenever A→α∣βA \rightarrow \alpha \; \vert \; \betaA→α∣β are two distinct productions of GGG, the following conditions hold:

  1. No left-recursion or ambiguity
  2. For no terminal aaa do both α\alphaα and β\betaβ derive strings beginning with aaa.
    • Equivalent to that FIRST(α)FIRST \lparen \alpha \rparenFIRST(α) and FIRST(β)FIRST \lparen \beta \rparenFIRST(β) are disjoint sets.
  3. At most one of α\alphaα and β\betaβ can derive the empty string.
  4. If β⇒∗ϵ\beta \xRightarrow{*} \epsilonβ∗​ϵ, then α\alphaα does not derive any string beginning with a terminal in FOLLOW(A)FOLLOW \lparen A \rparenFOLLOW(A), and vice versa.
    • Equivalent to that if ϵ\epsilonϵ is in FIRST(β)FIRST \lparen \beta \rparenFIRST(β), then FIRST(α)FIRST \lparen \alpha \rparenFIRST(α) and FOLLOW(A)FOLLOW \lparen A \rparenFOLLOW(A) are disjoint sets, and vice versa.

  1. For each terminal aaa in FIRST(α)FIRST \lparen \alpha \rparenFIRST(α), add A→αA \rightarrow \alphaA→α to M[A,a]M \lbrack A, a \rbrackM[A,a]
  2. If ϵ\epsilonϵ is in FIRST(α)FIRST \lparen \alpha \rparenFIRST(α), then for each terminal bbb in FOLLOW(A)FOLLOW \lparen A \rparenFOLLOW(A), add A→αA \rightarrow \alphaA→α to M[A,b]M \lbrack A, b \rbrackM[A,b]. # Table-driven predictive parsing

Model of a table-driven predictive parser

Image credit to Compilers: Principles, Techniques, and Tools 2nd Edition

  • Input: A string www and a parsing table MMM for grammar GGG.
  • Output: A leftmost derivation of www or an error indication.

Initially, the parser is in a configuration with w$w\$w$ in the input buffer and the start symbol SSS of GGG on top of the stack, above $\$$.

  • Set the input pointer ipipip to point to the first symbol of www
